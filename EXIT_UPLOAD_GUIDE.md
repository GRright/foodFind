# 🚀 离场上传策略实现指南

## 📋 核心概念：本地记账，离场结账

**核心思想**：用户在使用小程序时，所有个性化数据（点击、浏览、喜欢等）只写入本地存储；当用户退出小程序或切到后台时，才一次性批量上传到云端。

**效果**：用户点击20个菜，原本要调用20次云函数，现在只调用1次！

## 🎯 功能特性

### ✅ 1. 本地记账（使用时）
- **recordBehavior()** - 记录用户行为（点击、浏览等）到本地队列
- **updateProfile()** - 更新用户画像到本地
- **零延迟**：所有操作都是本地存储，瞬间完成
- **无网络依赖**：电梯里、信号差也能正常使用

### ✅ 2. 离场结账（退出时）
- **onHide 触发**：用户按Home键、锁屏、切换聊天窗口时自动触发
- **批量上传**：一次调用 `batchSync` 云函数，上传所有数据
- **包含内容**：
  - 行为队列（点击、浏览、喜欢等）
  - 用户画像更新
  - 会话摘要统计

### ✅ 3. 启动补传（兜底机制）
- **onLaunch 检查**：启动时检查是否有未上传的数据
- **静默补传**：自动上传上次未成功的数据
- **防止丢失**：即使用户手机突然关机，数据也不会丢

### ✅ 4. 心跳上传（双保险）
- **15分钟间隔**：长时间使用时自动触发上传
- **防止积压**：避免数据积压过多

### ✅ 5. 预加载包（智能优化）
- **云端计算**：退出时云端分析用户行为，生成推荐
- **本地缓存**：预加载包保存到本地（24小时有效）
- **秒开体验**：下次打开直接展示预计算的推荐

## 🔧 使用方法

### 在页面中记录行为

```javascript
// 获取 ExitUploadManager
const app = getApp();

// 记录浏览行为（只写本地，离场时批量上传）
app.globalData.ExitUploadManager.recordBehavior(
  'view',      // actionType: view, like, dislike, click
  'recipe',    // itemType: recipe, movie
  123,         // itemId
  { duration: 10 }  // details: 额外信息
);

// 更新用户画像（只写本地，离场时同步）
app.globalData.ExitUploadManager.updateProfile({
  taste_weights: { spicy: 0.8 },
  preferences: { cuisine: 'chinese' }
});
```

### 获取预加载推荐

```javascript
// 优先使用预加载包（云端退出时生成）
const preloadPackage = app.globalData.ExitUploadManager.getPreloadPackage();
if (preloadPackage && preloadPackage.recommendations) {
  // 使用预计算的推荐
  this.setData({ recipes: preloadPackage.recommendations });
}
```

## 🔐 数据安全

1. **本地加密**：敏感数据使用微信 Storage 加密存储
2. **队列上限**：行为队列最多保存50条，防止内存溢出
3. **自动清理**：上传成功后自动清空本地数据
4. **异常处理**：上传失败保留数据，下次启动时补传
