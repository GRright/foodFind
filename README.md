# 吃点啥！

一个基于微信小程序云开发的智能三餐推荐应用，采用"本地记账，离场结账"的低成本架构设计。

## 项目简介

为用户解决"今天吃点啥"的日常决策难题，提供智能化的饮食推荐、个性化口味学习、场景化用餐建议。

**核心特点**：
- 🚀 **极速体验**：本地计算推荐，零网络延迟
- 💰 **超低成本**：云函数调用减少 90%+
- 🧠 **智能学习**：基于用户行为实时调整推荐
- 📱 **离线可用**：弱网环境下依然流畅使用

## 技术栈

- **微信小程序云开发**
  - 云函数（按量付费，极低调用量）
  - 云数据库（存储用户画像、行为数据）
  - 云存储（可选，用于图片等资源）
- **前端**
  - 微信小程序原生开发
  - WXML / WXSS / JavaScript

## 核心架构：本地记账，离场结账

### 低成本设计

```
用户使用（前台）
    │
    ├── 点击菜品 ──► 本地存储（0 云函数调用）
    ├── 浏览详情 ──► 本地存储（0 云函数调用）
    ├── 喜欢/不喜欢 ──► 本地存储（0 云函数调用）
    └── 修改偏好 ──► 本地存储（0 云函数调用）
    │
    ▼ 用户退出/切后台（onHide）
    │
批量上传（1 次云函数调用）
    │
    ├── 行为队列（20条点击）
    ├── 用户画像更新
    └── 会话摘要统计
    │
    ▼ 云端处理
    │
    ├── 更新数据库
    ├── 分析用户偏好
    └── 生成预加载包（下次推荐）
```

### 成本对比

| 场景 | 传统架构 | 本项目架构 | 节省 |
|------|---------|-----------|------|
| 用户浏览20个菜 | 20次云函数调用 | 1次云函数调用 | **95%** |
| 月度云函数费用 | ¥50-100 | ¥5-10 | **90%** |

## 快速开始

### 1. 开通微信云开发

1. 打开微信开发者工具
2. 点击顶部菜单 **云开发**
3. 按提示开通云开发环境（新用户有免费额度）
4. 记录你的 **环境 ID**

### 2. 配置项目

修改 `miniprogram/app.js` 中的环境 ID：
```javascript
wx.cloud.init({
  env: '你的环境ID',  // 替换为你的云环境 ID
  traceUser: true
});
```

### 3. 创建数据库集合

在云开发控制台 → **数据库** 中，创建以下集合：
- `users` - 用户信息
- `user_behaviors` - 用户行为数据
- `recipes` - 菜谱数据（可选，默认使用本地数据）
- `movies` - 电影数据（可选，默认使用本地数据）
- `onboarding_answers` - 引导页答案

### 4. 上传云函数

在微信开发者工具中：
1. 右键 `cloudfunctions/login` → **上传并部署：云端安装依赖**
2. 右键 `cloudfunctions/batchSync` → **上传并部署：云端安装依赖**
3. 右键 `cloudfunctions/getOnboardingQuestions` → **上传并部署**
4. 右键 `cloudfunctions/saveOnboardingAnswers` → **上传并部署**
5. 右键 `cloudfunctions/checkOnboardingStatus` → **上传并部署**
6. 右键 `cloudfunctions/getRecipes` → **上传并部署**
7. 右键 `cloudfunctions/getMovies` → **上传并部署**

### 5. 编译运行

点击微信开发者工具的 **编译** 按钮，即可运行小程序。

## 核心功能

### 1. 智能推荐
- **场景感知**：根据时间（早/午/晚餐）、周末自动调整推荐
- **个性化学习**：基于用户喜欢/不喜欢实时调整权重
- **探索机制**：20% 探索率，避免信息茧房
- **预加载包**：退出时云端计算，下次秒开

### 2. 用户引导
- **首页弹窗**：非强制引导，用户可选择"稍后再说"
- **口味问卷**：辣度、菜系、烹饪方式等偏好设置
- **渐进式学习**：根据用户行为持续优化推荐

### 3. 行为记录
- **本地记录**：所有行为先写本地，零延迟
- **批量上传**：退出时一次性上传
- **实时反馈**：喜欢/不喜欢立即影响当前推荐

### 4. 数据同步策略
- **离场上传**：onHide 时批量同步
- **启动补传**：防止数据丢失
- **心跳上传**：15分钟间隔，防止积压
- **预加载包**：云端计算，本地缓存

## 推荐算法

### 分数计算
```
基础分：0.5
+ 用户画像匹配：+0.2
+ 场景匹配（早餐/周末）：+0.1~0.2
+ 实时行为权重：-0.5 ~ +0.3
= 最终得分（0-1）
```

### 探索机制
- 80% 个性化推荐（基于画像）
- 20% 探索推荐（随机多样化）
